**前后端分离要点：**

- **前端**：仅负责 UI、上传、目标调选择、结果展示、下载、前端日志；不包含 OCR/移调/图像处理。
- **后端**：提供 REST API，并托管前端构建产物（方案 A 单端口）；所有识别、移调、渲染在后端完成。
- **交互**：单张图片同步处理，`POST /api/transpose` 一次请求返回结果图 + JSON + warnings。

**说明：**

- `frontend/` 与 `backend/` 在**目录与职责**上完全分离：前端为 React + Tailwind，Vite 构建；后端为 FastAPI 与核心流水线。
- **方案 A（单端口）**：前端 `npm run build` 输出到 `frontend/dist`，后端 `frontend.mode = "build"` 时托管该目录并回退 SPA 路由到 `index.html`，同一端口提供页面与 `POST /api/transpose`。
- 开发命令统一使用 Makefile：`make dev`（后端开发）、`make prod`（后端生产）、`make web`（前端开发服务器）。
- 入口职责约定：`Makefile` 是团队对外入口；`backend/run.py` 是后端启动实现细节，仅用于调试启动参数/排查入口问题时直接调用。

## 3. 前端模块与文件对照（React + Vite）

| 模块名 | 职责 | 主要文件 |
|--------|------|----------|
| **Upload** | 选择/拖拽上传图片、输入目标调、发起 POST /api/transpose | `frontend/src/components/Upload.tsx` |
| **LoadingState** | 同步等待时展示「处理中」、失败与错误提示 | `frontend/src/components/LoadingState.tsx` |
| **ResultViewer** | 展示原图与结果图、高亮变化区域、warnings/低置信度提示 | `frontend/src/components/ResultViewer.tsx` |
| **DownloadPanel** | 下载结果图、解析 JSON、导出调试信息（含 request_id） | `frontend/src/components/DownloadPanel.tsx` |
| **logger** | 前端日志：用户操作、请求/响应摘要、错误；可与 request_id 关联 | `frontend/src/lib/logger.ts` |

**入口与构建：**

- `frontend/index.html`：Vite 入口 HTML。
- `frontend/src/main.tsx`：React 入口；`App.tsx` 为根组件与路由。
- 样式：`src/index.css` 中引入 Tailwind，组件内可使用 Tailwind 类名。
- 生产：`npm run build` 输出到 `frontend/dist`，由后端按方案 A 托管。

## 4. 后端模块与文件对照

| 模块名 | 步骤 | 职责 | 主要文件 |
|--------|------|------|----------|
| **api** | — | 同步接口 POST /api/transpose、页面路由、统一 JSON、超时与异常 | `backend/tuneai/api/routes.py` |
| **task_manager** | — | 单次请求完整生命周期（7步流水线）、并行调度、耗时与错误日志 | `backend/tuneai/core/task_manager.py` |
| **storage** | — | 临时存储原图/中间结果/输出图，完成后清理 | `backend/tuneai/core/storage.py` |
| **preprocess** | 第一步（本地） | 灰度化、去噪、倾斜校正（deskew），返回灰度 ndarray | `backend/tuneai/core/preprocess.py` |
| **vision_llm** | 第二步A（线上） | LangChain ChatOpenAI + OpenAI-compatible 接口，识别调号，输出主音字符串如 `G` | `backend/tuneai/core/vision_llm.py` |
| **ocr** | 第二步B（线上） | 基于 `ocr.provider + ocr.config` 路由 OCR 实现，输出全字符 bbox + 识别内容（`OcrChar` 列表） | `backend/tuneai/core/ocr/` |
| **filter** | 第三步（本地） | 过滤 OCR 结果，保留单字符 `0-7`，转换为 `NoteEvent` / `RestEvent` | `backend/tuneai/core/filter.py` |
| **music** | 第四步（本地） | 调号换算、12-TET 解码/编码、转调、等音策略；工具函数 `count_accidentals` / `shift_octave` | `backend/tuneai/core/music.py` |
| **validate** | 第五步（本地 + LLM） | 三层校验：规则 → LLM文本 → VL视觉；失败只记 warning，不阻断 | `backend/tuneai/core/validate.py` |
| **pitch_adjust** | 第六步（本地 + LLM） | 音高评估（过高降八度）与迭代换调（减少半音）；确定调用 LLM，**MVP 阶段暂不实现，原样透传** | `backend/tuneai/core/pitch_adjust.py` |
| **render** | 第七步（本地） | OpenCV 批量白色擦除 + Pillow 渲染新数字，导出 PNG bytes | `backend/tuneai/core/render.py` |
| **llm** | — | LangChain structured output：调号纠错、事件补全、音高评估 stub | `backend/tuneai/core/llm.py` |
| **logging** | — | loguru 结构化日志、request_id ContextVar、各阶段打点 | `backend/tuneai/logging_config.py` |

**数据与配置：**

- `backend/tuneai/schemas/request_response.py`：API 请求/响应体。
- `backend/tuneai/schemas/score_ir.py`：乐谱 IR（与执行方案 §7 一致）。
- `backend/tuneai/config.py`：超时、临时目录、LLM/OCR 等配置。

## 5. 前后端数据流（分离边界）

```
前端                                    后端
────                                    ────
1. 用户选择图片 + 目标调
2. 前端校验（格式、目标调枚举）
3. POST /api/transpose (multipart: image, target_key)
   ────────────────────────────────────────→
4. 生成 request_id，写入 response
5. 接收图片 → storage 临时保存
6. task_manager 调用 pipeline（7步）：
   [1] preprocess
   → [2] Qwen-VL (调号) ‖ 阿里OCR (全字符 bbox)（并行）
   → [3] filter → [4] music(转调) → [5] validate → [6] pitch_adjust → [7] render
7. 输出：output_image, score_json, warnings
8. 清理临时文件
   ←────────────────────────────────────────
9. 返回 JSON: { success, output_image, score_json, warnings, request_id }
10. 前端：loading 结束，展示 result-viewer
11. 前端：download-panel 提供下载与 request_id 调试
```

- **边界**：所有「识别 → 移调 → 渲染」都在后端完成；前端只发一张图+目标调，收结果图+JSON+warnings+request_id。
- **日志**：后端全链路带 request_id；前端日志可记录同一 request_id，便于前后端联调与问题排查。

## 6. 接口约定（前后端契约）

- **唯一业务接口**：`POST /api/transpose`
  - 请求：`multipart/form-data`，字段 `image`（文件）、`target_key`（如 `C`、`G`、`Eb`）。
  - 成功响应：`success: true`，`output_image`（base64 或 URL 依实现）、`score_json`、`warnings`、`processing_time_ms`、`request_id`。
  - 错误响应：`success: false`，`error_code`、`error_message`、`request_id`。
- **页面与静态资源**：方案 A 下由后端托管 `frontend/dist`，提供 `GET /`（SPA）、`GET /assets/*`，非 API 路径回退到 `index.html`。
- 暂不提供：任务队列、轮询、历史记录；前后端仅围绕「单图同步处理」协作。

## 7. 技术栈小结

| 层级 | 技术选型 | 说明 |
|------|----------|------|
| 前端 | **React + Tailwind CSS**，**Vite** 打包 | SPA，构建产物由后端托管（方案 A 单端口） |
| 后端 | FastAPI + Python | 同步处理，无队列、无数据库、无用户系统 |
| 调号识别 | **视觉 LLM（OpenAI-compatible）** | 输入整张图，仅识别调号，输出结构化 `1=X` |
| 字符 bbox | **阿里云 OCR** | 全字符识别，输出 bbox + 内容，本地过滤保留 0-7 |
| 图像处理 | OpenCV + Pillow + NumPy | 预处理（deskew）、像素回写 |
| 日志 | loguru + 前端 logger | 前后端均带 request_id，便于联调 |

以上架构与《TuneAI_完整执行方案》一致，前端采用 **React + Tailwind CSS + Vite**，前后端分离，接口与方案 A 单端口部署不变。
