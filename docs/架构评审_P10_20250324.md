### 🔴 P0-1：全局可变配置 — 测试地狱根源

**文件：** `backend/tuneai/config.py:143`

```python
_config: dict[str, Any] | None = None

def get_config() -> dict[str, Any]:
    global _config
    if _config is None:
        _config = load_config()
```

**问题：**
- `load_config()` 调用 `_overlay_provider_section()` 直接修改加载出的 dict

**修复方案：**
```python

class TuneAIConfig(BaseSettings):

    providers: dict[str, ProviderConfig]
    pipeline: PipelineConfig
    # ...

    class Config:
        env_prefix = "TUNEAI_"
        env_nested_delimiter = "_"

# 方案 B：Lifespan inject
# 在 main.py lifespan 中创建 config，通过 DI 传入各模块
```

**影响范围：** 所有依赖 config 的模块
**预估工时：** 4h

### 🔴 P0-2：LLM Client 缓存冻结运行时配置

**文件：** `backend/tuneai/core/adapters/llm.py:27`

```python
_llm_instances: dict[tuple[str, str, str], Any] = {}

# Key = (provider, model, base_url)
```

**问题：**
- 缓存命中时直接返回旧实例，config 变了但实例不更新
- 多 tenant 场景或运行时切换 provider 会得到 stale client
- 连接池资源无法优雅回收

**修复方案（三选一）：**

| 方案 | 优点 | 缺点 |
|------|------|------|
| A. 去掉缓存，每次请求创建新 client | 简单，config 即时生效 | 有连接开销 |
| B. `ClientPool` with TTL + refresh | 正确，兼顾性能 | 需要自己实现池 |
| C. Lifespan 管理 + DI 注入 | 最干净 | 改动面大 |

**推荐方案 C：** LLM Client 生命周期交给 `main.py` lifespan 管理，通过 dependency injection 注入 pipeline。

**影响范围：** `llm.py`, `vision.py`, `llm_client.py`, `routes.py`
**预估工时：** 6h

### 🟡 P1-1：OCR Prompt 输出未强约束 — 线上隐患

**文件：** `backend/tuneai/core/adapters/ocr/multimodal.py:14-28`

**问题：**
- Prompt 要求模型输出 JSON，但仅靠 `response_format: json_object` 无法完全保证
- 模型可能返回 markdown 代码块、额外字段、部分损坏的 JSON
- 当前 `_parse_ocr_response()` 有防御但不够健壮

**当前防御（不足）：**
```python
# multimodal.py:82-83
if not char.get("text"):
    continue  # 静默跳过无效字段
```

**修复方案：**
1. 增加 `max_tokens` 上限防止截断
2. `json.loads()` 失败时触发 fallback 重试（1次）
3. 考虑用 `json_schema` + `function_calling` 替代 free-form prompt

```python
# 建议的 fallback 逻辑
def _parse_ocr_response_with_fallback(raw: str, retries=1) -> list[OcrChar]:
    try:
        return _parse_ocr_response(raw)
    except (json.JSONDecodeError, ValueError) as e:
        if retries > 0:
            # 尝试清理 markdown code block
            cleaned = re.sub(r"```(?:json)?\n(.*?)\n```", r"\1", raw, flags=re.DOTALL)
            return _parse_ocr_response_with_fallback(cleaned, retries - 1)
        raise ValueError(f"OCR response parse failed after retries: {e}")
```

**影响范围：** `ocr/multimodal.py`
**预估工时：** 2h

```
### Task-3：OCR 回复加固

**Prompt：**

```
# 角色
你是 P8 后端工程师，负责加固 TuneAI 的 OCR 回复解析。

# 目标
`backend/tuneai/core/adapters/ocr/multimodal.py` 的 JSON 解析不够健壮，
模型可能返回 markdown 代码块、截断 JSON、多余字段。需要增加多层防御。

# 当前代码问题
位置：`backend/tuneai/core/adapters/ocr/multimodal.py`

当前 `_parse_ocr_response()` 函数的问题：
1. 直接 `json.loads()` 原始文本，无 markdown 清理
2. 字段验证不严格：跳过无效字段但不报错
3. 无重试机制

# 实现方案

## 1. 增强 `_safe_json_parse()`（新增）
```python
def _safe_json_parse(raw: str, retries: int = 1) -> dict:
    """带清理和重试的 JSON 解析"""
    # Step 1: 清理 markdown code block
    cleaned = re.sub(r"```(?:json)?\n(.*?)\n```", r"\1", raw, flags=re.DOTALL)
    cleaned = cleaned.strip()

    # Step 2: 尝试直接解析
    try:
        return json.loads(cleaned)
    except json.JSONDecodeError:
        pass

    # Step 3: 提取 JSON 对象（处理前后有文本的情况）
    match = re.search(r"\{[\s\S]*\}", cleaned)
    if match:
        try:
            return json.loads(match.group())
        except json.JSONDecodeError:
            pass

    # Step 4: 重试（截断修复）
    if retries > 0:
        # 取前半部分，通常是完整 JSON
        half = len(cleaned) // 2
        return _safe_json_parse(cleaned[:half], retries - 1)

    raise ValueError(f"Failed to parse OCR response after retries: {raw[:100]}")
```

## 2. 增强 `_parse_ocr_response()` 字段验证
```python
def _parse_ocr_response(data: dict) -> list[OcrChar]:
    chars = data.get("chars", [])
    result = []
    for i, char in enumerate(chars):
        # 严格验证字段
        if not isinstance(char, dict):
            continue
        text = char.get("text", "")
        if not isinstance(text, str) or not text.strip():
            continue  # 空文本跳过
        bbox = char.get("bbox")
        if not isinstance(bbox, list) or len(bbox) != 4:
            continue  # 无效 bbox 跳过
        confidence = char.get("confidence", 1.0)
        if not isinstance(confidence, (int, float)):
            confidence = 1.0
        result.append(OcrChar(text=text.strip(), bbox=bbox, confidence=confidence))
    if not result:
        raise ValueError("No valid characters in OCR response")
    return result
```

## 3. 在 `run_multimodal_ocr()` 中集成

## 3. 实现 `OcrOptimizedPreprocessor`（新增）
```python
class OcrOptimizedPreprocessor:
    """针对 OCR 优化的预处理策略"""
    CANONICAL_HEIGHT = 800  # 归一化目标高度

    def process(self, img: np.ndarray) -> np.ndarray:
        # Step 1: 分辨率归一化
        img = self._normalize_size(img)

        # Step 2: 去噪
        gray = cv2.cvtColor(img, cv2.COLOR_BGR2GRAY)
        denoised = cv2.fastNlMeansDenoising(gray)

        # Step 3: CLAHE 对比度增强
        clahe = cv2.createCLAHE(clipLimit=2.0, tileGridSize=(8, 8))
        enhanced = clahe.apply(denoised)

        # Step 4: 自适应二值化
        binary = cv2.adaptiveThreshold(
            enhanced, 255,
            cv2.ADAPTIVE_THRESH_GAUSSIAN_C,
            cv2.THRESH_BINARY,
            blockSize=11,
            C=2
        )

        # Step 5: 形态学操作去噪
        kernel = cv2.getStructuringElement(cv2.MORPH_RECT, (2, 2))
        cleaned = cv2.morphologyEx(binary, cv2.MORPH_OPEN, kernel)

        # Step 6: 纠偏
        return self._deskew(cleaned)

    def _normalize_size(self, img: np.ndarray) -> np.ndarray:
        h, w = img.shape[:2]
        if h > self.CANONICAL_HEIGHT:
            scale = self.CANONICAL_HEIGHT / h
            return cv2.resize(img, None, fx=scale, fy=scale, interpolation=cv2.INTER_AREA)
        return img
```
